# 网球社交平台项目介绍

## 一、项目概述

### 1.1 项目名称
网球社交平台 (Tennis Social Platform)

### 1.2 项目定位
一款专注于网球爱好者的移动社交应用，为用户提供动态分享、约球组局等核心功能，旨在构建活跃的网球社交生态圈。

### 1.3 目标用户
- 网球爱好者
- 业余网球选手
- 网球俱乐部成员
- 寻找球友的运动爱好者

## 二、核心功能

### 2.1 动态发布模块
**功能描述：**
- 用户可以发布网球相关的图文动态
- 支持上传照片、视频
- 添加位置标签（球场位置）
- 话题标签功能（#技术分享、#装备推荐等）
- 点赞、评论、转发互动

**使用场景：**
- 分享打球瞬间
- 晒出比赛成绩
- 交流技术心得
- 推荐球场和装备

### 2.2 约球功能模块
**功能描述：**
- 发起约球活动
  - 设置时间、地点
  - 设定参与人数
  - 水平要求说明
  - 费用分摊设置
- 约球广场
  - 浏览附近的约球信息
  - 筛选功能（时间、地点、水平）
  - 快速报名参与
- 约球管理
  - 参与者管理
  - 活动提醒
  - 活动取消/修改

**使用场景：**
- 周末找球友一起打球
- 组织小型业余比赛
- 寻找相同水平的练球伙伴
- 约教练或陪练

### 2.3 用户系统
**功能描述：**
- **用户注册/登录**
  - 手机号验证码登录
  - 一键登录（老用户快速登录）
  - 国家/地区代码选择
  - 自动Token刷新
- **个人信息管理**
  - 昵称、头像
  - 网球水平标注（初级/中级/高级/专业）
  - 擅长打法（底线型/上网型/全场型）
  - 常去球场
  - 球拍品牌
- **个人主页**
  - 动态列表
  - 约球历史
  - 粉丝/关注
  - 个人统计
- **消息通知**
  - 系统通知
  - 互动消息
  - 约球邀请
  - 登录提醒

### 2.4 社交互动
**功能描述：**
- 关注/粉丝系统
- 私信聊天
- 评论互动
- 点赞收藏

## 三、技术架构

### 3.1 前端技术栈
- **框架：** Flutter
- **语言：** Dart
- **状态管理：** Riverpod
- **优势：**
  - 真正的跨平台（iOS/Android/Web/Desktop）
  - 接近原生的高性能
  - 丰富的 Material Design 组件
  - 热重载开发体验
  - 一套代码多端运行

### 3.2 后端技术栈
- **框架：** FastAPI (Python)
- **优势：**
  - 高性能异步处理
  - 自动生成API文档
  - 类型检查和验证
  - 开发效率高

### 3.3 数据库
- **数据库：** PostgreSQL
- **优势：**
  - 强大的关系型数据库
  - 支持复杂查询
  - 地理位置数据支持（PostGIS）
  - 事务处理可靠

### 3.4 部署平台
- **云服务商：** 阿里云
- **容器化：** Docker + Docker Compose
- **服务组件：**
  - ECS 云服务器（Docker 部署）
  - PostgreSQL 数据库（Docker 容器）
  - OSS 对象存储（图片/视频）
  - Nginx 反向代理（Docker 容器）
  - CDN 内容分发

## 四、核心数据模型

### 4.1 用户表 (users)
- id: 用户唯一标识
- phone: 手机号（主要登录凭证）
- country_code: 国家/地区代码
- username: 用户名（可选）
- nickname: 昵称
- avatar: 头像URL
- skill_level: 技术水平（beginner/intermediate/advanced/professional）
- play_style: 打法风格（baseline/serve_and_volley/all_court）
- bio: 个人简介
- is_phone_verified: 手机号验证状态
- is_profile_completed: 资料完善状态
- created_at: 创建时间
- last_login_at: 最后登录时间

### 4.2 动态表 (posts)
- id: 动态ID
- user_id: 发布者ID
- content: 动态内容
- images: 图片URL数组
- location: 位置信息
- tags: 标签
- likes_count: 点赞数
- comments_count: 评论数
- created_at: 发布时间

### 4.3 约球活动表 (matches)
- id: 活动ID
- creator_id: 创建者ID
- title: 活动标题
- description: 活动描述
- location: 球场位置
- scheduled_time: 约球时间
- max_participants: 最大人数
- skill_requirement: 水平要求
- cost: 费用
- status: 状态（进行中/已满/已结束）
- created_at: 创建时间

### 4.4 评论表 (comments)
- id: 评论ID
- post_id: 动态ID
- user_id: 评论者ID
- content: 评论内容
- parent_id: 父评论ID（支持回复）
- created_at: 评论时间

### 4.5 约球参与表 (match_participants)
- id: 记录ID
- match_id: 活动ID
- user_id: 参与者ID
- status: 状态（已报名/已确认/已取消）
- joined_at: 加入时间

## 八、成功关键因素

1. **用户体验至上：** 简洁直观的界面，流畅的操作体验
2. **社区运营：** 积极的社区氛围营造，优质内容引导
3. **功能实用性：** 真正解决用户痛点，而非功能堆砌
4. **技术稳定性：** 确保系统稳定运行，快速响应
5. **市场推广：** 精准的用户获取策略，口碑传播

---

**项目愿景：** 成为网球爱好者的首选社交平台，让每个人都能轻松找到球友，享受网球运动的乐趣！
